## 2024-03-22 - [Redundant Process Executions and Duplicate Network Polling]
**Learning:** Found two major inefficiencies that severely impact performance without triggering obvious errors. In `clipper.py`, `get_video_info` runs `ffprobe` as a subprocess for every clip generated (up to 8 times per video). Since the video file doesn't change during clipping, this blocks the worker thread and wastes CPU time unnecessarily. In `static/script.js`, HTTP polling executes blindly every 2 seconds via `setInterval` even when the application is actively streaming progress updates over an established WebSocket connection.
**Action:** Always memoize/cache operations that query static files or hardware information inside loops/iterators using `@functools.lru_cache`. Additionally, always implement early exit conditions in fallback mechanisms (like polling) when the primary real-time channel (WebSockets) is active.
